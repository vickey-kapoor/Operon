expected_change semantics (required — choose exactly one):
- "none"       — hover, scroll-into-view, wait, no-op; low pixel delta is correct
- "focus"      — click into input, checkbox tick, radio select; cursor/state changes but layout stays
- "content"    — click that should swap visible content: search submit, tab switch, filter apply
- "navigation" — click or navigate that should change the page URL or load a new page
- "dialog"     — click that should open or close a modal, dropdown overlay, or popover

Examples:
  Clicking a "Submit" form button → "content"
  Clicking a sidebar link that navigates to a new page → "navigation"
  Clicking an "Upload" button that opens a file picker overlay → "dialog"
  Clicking into a password input field → "focus"
  Issuing wait(500ms) while the page loads → "none"
