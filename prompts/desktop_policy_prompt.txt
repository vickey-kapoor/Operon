Action type details:
- click: Click at screen coordinates. Requires x, y. Use target_element_id if a matching element was reported in perception.
- double_click: Double-click at screen coordinates. Requires x, y. Use for opening files, selecting words, or activating items that need a double-click.
- right_click: Right-click at screen coordinates. Requires x, y. Use to open context menus on desktop, files, taskbar icons, etc.
- drag: Drag from one position to another. Requires x, y (start) and x_end, y_end (destination). Use for moving windows, dragging files, resizing elements, or rearranging items.
- scroll: Scroll at a position. Requires scroll_amount (positive=up, negative=down, e.g. 3 or -3) and x, y (where to scroll). Use for scrolling through documents, web pages, lists, or file explorers.
- hover: Move the mouse to a position without clicking. Requires x, y. Use to reveal tooltips, trigger hover menus, or inspect elements before acting.
- type: Click at coordinates to focus, then type text. Requires text, x, y.
- press_key: Press a single key (e.g. "enter", "tab", "escape", "backspace"). Requires key.
- hotkey: Press a key combination (e.g. "win+r", "ctrl+c", "ctrl+shift+esc"). Requires key with "+" separated keys.

- screenshot_region: Capture a specific rectangular region of the screen. Requires x, y (top-left) and x_end, y_end (bottom-right). Use for focused analysis of a specific area.
- wait: Wait for a specified duration. Provide wait_ms (1-30000).
